# Release Delivery Checklist

기획팀 프롬프트에 누락이 있어도 개발팀이 릴리즈 산출물을 보완 입력하기 위한 표준 체크리스트.

## 1) 입력 누락 보완란
- 버전: 
- 패치명: 
- 메인 HTML 대상 파일: 
- 중요도(경미/중간/중요): 
- 프롬프트 누락 항목(있으면): 
- 개발팀 보완 입력 내용: 

## 2) 필수 산출물 (기본 3종 + 조건부 1종)
- [ ] 메인 HTML(`index.html` 또는 단일 메인 HTML) 변경 반영
- [ ] `docs/releases/EvoSim_latest.html` 동기화
- [ ] `docs/releases/archive/EvoSim_<version>_<patch>.html` 스냅샷 생성
- [ ] `docs/releases/EvoSim_CHANGELOG.txt` 갱신 (중간/중요 패치인 경우에만)

## 3) 실행보고 고정 규격
- [ ] 변경내용(10문장 이내)
- [ ] 프롬프트와 다르게 변경한 사항(4문장 이내, 있으면 작성)
- [ ] 주석변경사항(2문장 이내, 있으면 작성)
- [ ] 권고사항(1문장 이내, 있으면 작성)
- [ ] 임시디버그 도입 여부(패치별: 예/아니오)
- [ ] 임시디버그 제거 여부(패치별: 예/아니오)

## 4) 브랜치 규칙 검증
- [ ] 버전 증가 작업 브랜치가 `release/<version>-<task>` 형식임
- [ ] 동일 버전 작업 누적 브랜치 원칙을 준수함

## 5) 머지 게이트 통과 확인
- [ ] PR 템플릿 체크박스와 본 체크리스트가 일치함
- [ ] 최종 리뷰에서 누락 항목 없음
